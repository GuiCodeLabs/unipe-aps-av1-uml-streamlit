# :microphone: Roteiro de Apresentação

## :pushpin: 1. Abertura

Boa noite, professor.

Nós desenvolvemos a atividade da disciplina de **Análise e Projeto de Sistemas**, organizada em um único repositório no GitHub, contendo a parte de análise, diagramas e aplicações em Python com Streamlit.

O trabalho foi estruturado para atender às exigências da atividade, separando cada questão em análise, diagramas e aplicação.

---

## :open_file_folder: 2. Organização do projeto

O projeto foi dividido nas seguintes partes:

- `docs/` para os arquivos de apoio e enunciado
- `extras/` para materiais complementares
- `questoes/` para as 11 questões da atividade

Dentro de cada questão, organizamos:
- `analise/`
- `diagramas/`
- `app/`
- `README.md`

Essa organização foi escolhida para facilitar a leitura, manutenção e apresentação do trabalho.

---

## :books: 3. Parte de análise

Na parte de análise, para cada questão, identificamos:

- as classes do problema
- os atributos de cada classe
- os métodos principais
- os relacionamentos entre as classes

Também criamos separadamente:
- requisitos funcionais
- requisitos não funcionais

Essa etapa foi feita com base no cenário descrito em cada exercício.

---

## :triangular_ruler: 4. Diagramas de classes

Depois da análise, modelamos os diagramas de classes.

Em cada diagrama, representamos:
- classes
- atributos
- métodos
- relacionamentos
- multiplicidades, quando exigidas no enunciado

Os diagramas foram produzidos em PlantUML, permitindo manter o arquivo-fonte e também gerar a imagem final para visualização.

---

## :computer: 5. Aplicações desenvolvidas

Na parte prática, desenvolvemos aplicações em **Python com Streamlit**, como solicitado.

Cada aplicação busca representar o problema da questão em funcionamento, com interface simples e recursos como:
- cadastro
- listagem
- edição
- exclusão
- consultas
- relatórios

Nas questões em que fez sentido, utilizamos banco de dados para persistência das informações.

---

## :bulb: 6. Exemplo de funcionamento

Aqui podemos abrir uma ou duas questões como exemplo.

### Exemplo de fala:
“Vou mostrar rapidamente uma das aplicações desenvolvidas.  
Nela é possível visualizar a interface, cadastrar dados, consultar informações e verificar como a modelagem da análise foi transformada em uma aplicação funcional.”

Nesse momento, apresentar:
- a pasta da questão
- o arquivo de análise
- o diagrama de classes
- a aplicação em execução

---

## :dart: 7. Decisões do projeto

Algumas decisões importantes do trabalho foram:

- manter padronização entre todas as questões
- separar análise, diagrama e aplicação por pasta
- criar README para facilitar a navegação
- utilizar Streamlit por ser uma forma prática de construir interfaces web em Python
- manter todos os artefatos no mesmo repositório, como pedido pelo professor

---

## :white_check_mark: 8. Resultado final

Ao final, entregamos:

- as 11 questões analisadas
- os diagramas de classes
- os requisitos funcionais e não funcionais
- as aplicações em Python com Streamlit
- toda a documentação organizada no GitHub

---

## :handshake: 9. Encerramento

Com isso, concluímos a atividade buscando atender integralmente ao que foi solicitado, tanto na parte de análise orientada a objetos quanto na parte prática de desenvolvimento das aplicações.
